## 📈 工作原理

### 缓存层次结构

```
┌─────────────────────────────────────┐
│  用户 UI 层                           │
│  (搜索、筛选、排序 - 毫秒级)         │
└──────────────┬──────────────────────┘
               │
┌──────────────▼──────────────────────┐
│  @st.cache_data 缓存层               │
│  build_chain_master_v2()             │
│  (第1次: 15-30秒, 后续: <1秒)       │
└──────────────┬──────────────────────┘
               │
┌──────────────▼──────────────────────┐
│  @st.cache_data 缓存层               │
│  load_chain_data_v2()                │
│  (第1次: 5-10秒, 后续: <100ms)      │
└──────────────┬──────────────────────┘
               │
┌──────────────▼──────────────────────┐
│  数据引擎                            │
│  (读取源文件、数据库)               │
└─────────────────────────────────────┘
```

### 使用流程

```
用户访问"缺货全链路追踪"
         │
         ├─ 首次访问 (缓存未命中)
         │  ├─ load_chain_data_v2() → 从文件加载 (5-10秒)
         │  ├─ build_chain_master_v2() → 全链路关联 (10-20秒)
         │  └─ 保存到缓存 (TTL=1小时)
         │
         └─ 后续访问 (缓存命中)
            ├─ load_chain_data_v2() → 从缓存读取 (<100ms)
            ├─ build_chain_master_v2() → 从缓存读取 (<100ms)
            └─ 总耗时 <1秒 ⚡

用户进行操作:
  ├─ 搜索 → pandas 字符串匹配 (内存中) → <100ms ⚡
  ├─ 筛选 → boolean mask 操作 (内存中) → <100ms ⚡
  ├─ 排序 → pandas 排序 (内存中) → <100ms ⚡
  └─ 展开/收起 → DOM 操作 (浏览器中) → <100ms ⚡
```
