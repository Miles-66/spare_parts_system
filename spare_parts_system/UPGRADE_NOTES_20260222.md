## 一、合同数据源处理

### 1.1 数据源
- **文件名**: `dayu可用的进出口备件合同明细`
- **目录**: `data_source/logistics/`
- **记录数**: 25,650 行

### 1.2 关联主键
- **复合主键**: 箱号 + 物料号
- **清洗方式**: 使用 `strict_clean()` 函数
  - 去空格
  - 去前导零
  - 去 `.0` 后缀

### 1.3 清洗效果验证
测试用例全部通过：
```

✅ strict_clean('0123456') = '123456'
✅ strict_clean('123.0') = '123'
✅ strict_clean('  001234.0  ') = '1234'
```

## 二、字段抓取

### 2.1 从合同明细表抓取的字段

| 字段名 | 原始列名 | 说明 |
|--------|----------|------|
| 合同号 | `合同编号 (进出口/香港备件合同) (备件合同)` | 合同编号 |
| 合同日期 | `创建时间` | 合同创建日期 |
| 发车申请单号 | `进出口备件发车申请单号` | 发车申请单号 |
| 预计到港日期 | `预计到达日期 (进出口备件发车申请单号) (备件发车申请)` | ETA |
| 到港地点 | `目的港 (进出口备件发车申请单号) (备件发车申请)` | 目的港 |
| 发运方式 | `发运方式 (进出口备件发车申请单号) (备件发车申请)` | 海运/空运等 |
| 物流运输单号 | `物流运输单号 (进出口备件发车申请单号) (备件发车申请)` | 物流单号 |

### 2.2 说明
- **合同表和发车申请表在同一文件中**，因此一次性抓取所有字段
- **去重逻辑**: 同一个箱号+物料号如果出现在多个合同中，取**最新的合同号**（按合同创建日期排序）

## 三、关联逻辑

### 3.1 关联流程
```
缺货表
  ↓ (物料号 + 客户订单号)
采购表
  ↓ (SAP订单号 + 物料号)
装箱表
  ↓ (箱号 + 物料号) ← 【本次升级重点】
合同表
  ↓ (包含发车信息)
物流在途
```

### 3.2 关联方式
- **关联类型**: 左关联（以缺货表为基准）
- **关联键**: `(箱号, 物料号)` 复合主键
- **清洗规则**: 两端都使用 `strict_clean()` 确保 100% 对齐

### 3.3 空值处理
- 若某箱货还没做合同 → 显示 "📋 待签合同"
- 若有合同但无发车信息 → 显示 "📜 合同已封，待发车"
- 若有预计到港日期 → 显示 "🚢 物流在途 (预计 {ETA} 到港)"

## 八、使用说明

### 8.1 查看物流进度
1. 进入 **缺货全链路追踪** 页面
2. 选择 **📊 全局监控大表** 视图
3. 查看表格中的物流相关列：
   - 合同号
   - 合同日期
   - 发车申请单号
   - 预计到港日期（带状态图标）
   - 到港地点

### 8.2 到港提醒
- 当某订单的预计到港日期在3天内，该行的 **预计到港日期** 列会显示：
  ```
  ❗🚢 2026-02-25 (还有3天)
  ```
- 建议优先处理这些即将到港的订单

### 8.3 状态说明
| 状态 | 含义 | 下一步行动 |
|------|------|-----------|
| 📋 待签合同 | 已装箱，但未签合同 | 跟进合同部门 |
| 📜 合同已封，待发车 | 已签合同，但未发车 | 跟进物流部门 |
| 🚢 物流在途 | 已发车，在途中 | 等待到港 |

## 九、注意事项

### 9.1 数据质量要求
- **箱号格式**: 必须清洗前导零和空格
- **物料号格式**: 必须清洗前导零、.0 后缀和空格
- **合同日期格式**: 自动转换为 datetime 格式

### 9.2 已知限制
- **预计到港日期覆盖率较低** (0.5%)
  - 原因：部分合同记录中该字段为空
  - 建议：完善合同数据录入

### 9.3 性能优化
- 使用字典匹配代替多次 merge，提升关联速度
- 预处理阶段完成数据清洗，避免重复计算
